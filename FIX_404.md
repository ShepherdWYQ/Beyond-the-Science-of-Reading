# 快速修复 404 错误

## 🚨 立即检查这 3 项

### 1. 确认 GitHub Pages 已启用

**操作步骤**：
1. 访问：`https://github.com/ShepherdWYQ/Beyond-the-Science-of-Reading/settings/pages`
2. 在 `Source`（源）部分，**必须选择**：`GitHub Actions`
3. 点击 `Save`（保存）

**如果显示 "Not configured"**：
- 选择 `GitHub Actions`
- 保存设置
- 等待 5-10 分钟

### 2. 检查 GitHub Actions 状态

**操作步骤**：
1. 访问：`https://github.com/ShepherdWYQ/Beyond-the-Science-of-Reading/actions`
2. 查看最新的工作流运行：
   - ✅ **绿色** = 成功（等待 2-3 分钟让网站生效）
   - ❌ **红色** = 失败（点击查看错误，然后告诉我）
   - 🟡 **黄色** = 进行中（请等待）

### 3. 确认网站地址

**正确的地址**：
```
https://ShepherdWYQ.github.io/Beyond-the-Science-of-Reading
```

**注意**：
- 必须包含项目名称 `/Beyond-the-Science-of-Reading`
- 大小写必须完全匹配
- 首次部署需要等待 5-10 分钟

## 🔧 如果仍然 404，尝试以下操作

### 操作 A：手动触发部署

1. 访问：`https://github.com/ShepherdWYQ/Beyond-the-Science-of-Reading/actions`
2. 点击左侧的 "Deploy Jekyll site to Pages"
3. 点击右侧的 "Run workflow" 按钮
4. 选择 `main` 分支
5. 点击 "Run workflow"
6. 等待构建完成（约 5 分钟）

### 操作 B：检查并修复配置

如果 GitHub Actions 显示构建失败，请：

1. **查看错误日志**：
   - 点击失败的工作流
   - 查看 "Build with Jekyll" 步骤的错误信息

2. **常见错误修复**：
   - 如果提示 `bundle install` 失败：确保 `Gemfile` 已提交
   - 如果提示 `Jekyll build` 失败：检查 `_config.yml` 语法

3. **重新推送**（如果需要）：
   ```bash
   git add .
   git commit -m "Fix configuration"
   git push origin main
   ```

### 操作 C：清除缓存并重试

1. **清除浏览器缓存**：
   - Chrome/Edge: `Ctrl+Shift+Delete` (Windows) 或 `Cmd+Shift+Delete` (Mac)
   - 或使用隐私模式访问网站

2. **等待更长时间**：
   - 首次部署可能需要 10-15 分钟
   - DNS 传播可能需要额外时间

## 📋 快速检查清单

在告诉我问题前，请确认：

- [ ] GitHub Pages 已启用（Settings → Pages → Source = GitHub Actions）
- [ ] GitHub Actions 显示绿色（构建成功）
- [ ] 等待了至少 5 分钟
- [ ] 使用了正确的 URL（包含项目名称）
- [ ] 清除了浏览器缓存

## 🆘 如果以上都不行

请告诉我：

1. **GitHub Actions 的状态**：
   - 是绿色、红色还是黄色？
   - 如果是红色，错误信息是什么？

2. **GitHub Pages 设置**：
   - Source 选择的是什么？（应该是 GitHub Actions）

3. **访问的 URL**：
   - 您访问的具体地址是什么？

4. **时间**：
   - 您推送代码后等了多长时间？

---

**大多数情况下，问题是因为 GitHub Pages 未启用或选择了错误的源。请先确认步骤 1。**
