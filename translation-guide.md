# 翻译规范指南

本文档详细说明了本翻译项目的翻译规范和注意事项。

## 基本原则

### 1. 保持原文结构

* 保持原文的段落结构
* 保持标题层级（H1, H2, H3 等）
* 保持列表格式（有序列表、无序列表）
* 保持代码块格式

### 2. 术语统一

* 所有术语翻译必须参考 `glossary.md` 中的术语对照表
* 首次出现的重要术语可在括号中标注英文原文
* 保持术语翻译的一致性

### 3. 代码保持原样

* 代码示例保持英文，不翻译
* 命令行命令保持英文
* 变量名、函数名、类名等保持英文
* 注释可以翻译，但需保持代码可运行

### 4. 图片路径

* 所有图片引用路径必须正确指向 `images/` 目录
* 图片路径格式：`![描述](images/chapter-x/image-name.png)`
* 确保图片文件存在于对应目录

### 5. 格式规范

* 遵循 Markdown 格式规范
* 标题前后保留空行
* 列表前后保留空行
* 代码块前后保留空行

### 6. 校对审核

* 翻译完成后进行自我校对
* 检查错别字、语法错误
* 检查术语一致性
* 检查格式规范

## 翻译技巧

### 1. 理解上下文

* 充分理解原文含义
* 注意技术概念的准确性
* 保持技术文档的专业性

### 2. 语言表达

* 使用流畅的中文表达
* 避免生硬的直译
* 保持技术文档的简洁性

### 3. 专业术语

* 优先使用业界通用术语
* 参考相关技术文档
* 保持术语的一致性

## 文件结构

### 章节文件

章节文件应放在 `chapters/` 目录下，命名格式：

* 核心章节：`Chapter X_ Title.md`
* 附录章节：`Appendix X_ Title.md`

### 原文文件

原文文件应放在 `original/` 目录下，保持相同的命名格式。

### 图片文件

图片文件应放在 `images/` 目录下，按章节组织：

```
images/
├── chapter-1/
│   └── image-1.png
├── chapter-2/
│   └── image-2.png
└── ...
```

## 常见问题

### Q: 如何翻译技术术语？

A: 参考 `glossary.md` 中的术语对照表，保持一致性。

### Q: 代码注释需要翻译吗？

A: 可以翻译，但需确保代码仍可正常运行。

### Q: 如何处理英文缩写？

A: 首次出现时给出中文全称和英文缩写，后续可直接使用缩写。

### Q: 图片引用路径如何设置？

A: 使用相对路径，格式：`![描述](images/chapter-x/image-name.png)`

- [ ] 术语翻译是否统一

- [ ] 格式是否符合规范
- [ ] 是否有错别字或语法错误
- [ ] 是否更新了 `progress.md`
